CoreData的简单使用

发表于 2014-07-29   |   分类于 iOS   |  

NSEntityDescription

上一章讲过，对数据的所有操作都是在NSManagedObjectContext（被管理对象上下文）上进行的，那究竟该如何进行操作。
既然要进行操作，那就需要有被操作的对象，而该对象便是“实体”（Entity）
什么实体。一般来说，实体是oc中的数据型类在CoreData中的映视，简单的讲，就是oc中Model型的类，在CoreData中便称为实体
NSEntityDescription的对象便是实体，每个实体都有2个重要的特性，一个是实体的名称：name，另一个与该实例相互联系的类 ：Class

实体由Attributes和Relationships组成
打开后缀为.xcdatamodeld的文件,点击左下方的Add Entity 按键，便可以新建一个新的实体，点击实体，便可以看到Attributes和Relationships


其中,Attributes类似于oc类中的属性，而Relationships则是不同类之间的联系

例如：建立一个名为Student的实体 有一个Attributes：name，一个Relationships：studentID
建立一个名为StudentID的属性 一个Attributes：ID ，一个Relationships：student


这样就在两个实体之间建立了联系，这是简单的一对一联系，可以通过一个实体访问另外一个实体
点击屏幕右下角的Editor Style 按钮


便可进入另一种编辑模式，可以明显看到两个类之间一条双箭头的线联系。

（ps：点击Editor中的Add Model Version，可以建立实体对应的类）

CoreData的基本操作

添加数据

添加数据的步骤

  1. 获得实体的名称和“被管理对象的上下文”
  2. 通过实体名称向该实体中插入一个NSManagedObject的实例对象（实体的每一实例对象都继承于NSManagedObject）
  3. 类似字典，通过键值的方式设置对象的objc属性（此时修改的数据仅仅是在“被管理对象的上下文”上，数据库中的数据没有被修改）
  4. 将NSManagedObjectContext的对象保存（数据库中的数据被修改）

删除数据

步骤：

  1. 获得需要删除的数据对象
  2. 将该数据从“被管理对象的上下文”中删除
  3. 保存数据

查找数据

  1. 建立一个数据请求
  2. 通过“被管理对象上下文”发送请求
  3. 返回数据，是一个数组

其中NSFetchRequest是一个设置请求数据的类
主要有3个属性

  1. entity：即设置向哪个实体请求数据
  2. predicate：设置请求的内容，通过谓词设置 例如“point BEGINSWITH ‘9’”
  3. sort :返回数据的排列方式
